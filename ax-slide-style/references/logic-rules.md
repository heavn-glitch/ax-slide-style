# 논리 전개 · 장표 표현 규칙

이 규칙은 슬라이드의 "내용"을 어떻게 배치할지 결정한다. 디자인 토큰(design-tokens.md)과 함께 항상 적용한다.

## 슬라이드 단위 규칙
1. **1페이지 1메시지.** 한 장에 주장 하나. 두 개면 두 장으로 나눈다.
2. **제목 = 결론.** `<h2>`에 그 장의 핵심 주장을 그대로 쓴다. 제목만 읽어도 논지가 통하게.
3. **펀치라인만 강조.** 제목 안에서 가장 중요한 단어구만 `<span class="accent">`로 감싼다(장당 1회).
4. **눈썹라벨은 분리.** `<p class="meta">`에 영문 카테고리(Why / Method / Compare…)를 둔다. 메시지가 아니라 위치표식이다.
5. **본문은 한 문장.** 카드·셀의 `<p>`는 포인트당 한 문장. 문단·중첩 불릿 금지.
6. **병렬 구조.** 분할 항목은 문법 형태를 평행하게 맞춘다(예: 반복 설명 / 반복 실수 / 반복 맥락).
7. **근거 표기.** 외부 사실을 인용하면 `<div class="source-line">Source: …</div>`.
8. **페이지 번호 고정.** 모든 슬라이드 우하단 `<div class="slide-no">N</div>`.

## 덱 전체 흐름 (why → what → how → so-what)
문제 제기(왜 필요한가) → 정의(무엇인가) → 해부/구조(어떻게 생겼나) →
방법(단계) → 실증(실제 예시·프롬프트) → 안전/주의 → 확장(그래서 어디까지).
- 섹션이 바뀌는 지점에는 **간지 슬라이드**(큰 단어 하나, `.quote` + `.center-stage`)를 끼워 호흡을 끊는다.
- 개념 설명 뒤에는 반드시 **실증 슬라이드**(실제 프롬프트·산출물)로 착지한다.

## 콘텐츠 → 템플릿 매핑
| 담을 내용 | 쓰는 컴포넌트 |
|---|---|
| 개념 3~4개 병렬 | `.grid-3` / `.grid-4` + `.cell`(아이콘+h3+한 문장) |
| A vs B 비교 | `.mini-table.row-3` / `.row-4` (첫 열·행 `.head`) |
| 순서 있는 절차 | `.timeline` + `.timeline-step`(01~05) |
| 실제 프롬프트/코드 | `.code-block` (+ `.prompt-label`) |
| 하나의 강조 결론 | `.callout` |
| 섹션 전환 | `.quote` + `.center-stage` |
| 도입/표지 | `<h1>` + `.lead`, `bg-hero` |

## 배경 리듬
`bg-dark`(기본) · `bg-soft` · `bg-hero`(도입/간지)를 쓰되, 톤 전환이 필요한 장은 `bg-light`(반전)로.
같은 배경을 3장 이상 연속 쓰지 않는다.
