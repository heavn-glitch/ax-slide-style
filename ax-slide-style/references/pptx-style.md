# PPTX 스타일 규격

PPTX를 만들 때 현재 환경의 `Presentations` 스킬과 `@oai/artifact-tool`을 사용하고 아래 스타일을 적용한다.

## 색

| 이름 | 값 | 용도 |
|---|---|---|
| ink | `#0A0908` | dark 배경 |
| inkSoft | `#15130F` | soft/hero 배경과 다크 카드 |
| inkRise | `#1F1C16` | 경계선 |
| bone | `#F0EDE5` | 제목과 주요 텍스트 |
| boneDim | `#C9C5BC` | 본문 |
| boneMute | `#7A766E` | 눈썹라벨·출처·번호 |
| cardBone | `#E6DFD3` | light 카드 |
| ember | `#EBA8FF` | 핵심 액센트 |
| emberDeep | `#C873E0` | 보조 글로우 |

light 배경에서는 제목을 ink, 본문을 `#3A362F`, 보조 텍스트를 `#6E6A62`로 반전한다.

## 캔버스와 타이포

- 비율: 16:9
- 캔버스: 13.333 × 7.5in
- 좌우 안전 여백: 0.62in
- 폰트: Pretendard, 코드만 Consolas
- hero 제목: 52pt 이상
- divider 제목: 60pt 이상
- 일반 제목: 35pt 이상
- 소제목·카드 제목: 24pt 이상
- 본문: 16pt 이상
- 눈썹라벨: 11pt, 대문자, 굵게, 자간 확대
- 출처: 9pt
- 페이지 번호: 10pt

문장을 줄이는 것을 글꼴 축소보다 우선한다. 제목은 의도하지 않게 두 줄로 넘어가지 않게 한다.

## 고정 골격

- 눈썹라벨: 상단 좌측
- 제목: 눈썹라벨 아래, 결론형 문장
- 본문: 제목 아래의 단일 아키타입
- 출처: 좌하단
- 페이지 번호: 우하단

제목은 run 배열로 구성하고 펀치라인 한 곳만 ember로 처리한다.

## 아키타입

### hero

중앙 정렬 표지. 큰 제목, 한 문장 리드, 선택적 발표자·대상 정보만 둔다.

### divider

섹션 전환. 큰 단어 또는 짧은 메시지 하나만 둔다.

### grid

3~4개의 병렬 개념. 카드마다 소제목과 한 문장 본문을 둔다.

### table

비교·매핑용. 3~4열을 권장하고 첫 행과 첫 열을 명확히 구분한다.

### timeline

최대 5단계 절차. 번호, 단계명, 한 문장 설명을 사용한다.

### code

프롬프트·코드 원문. 앰버 테두리의 다크 박스와 짧은 라벨을 사용한다.

### callout

핵심 결론 한 문장. 앰버 틴트 박스 하나만 사용한다.

## 배경

- dark: ink
- soft: inkSoft + 낮은 투명도의 ember 글로우
- hero: inkSoft + 양쪽 상단 ember 글로우
- light: bone + 반전 텍스트

논리 전환에 맞춰 사용한다. 단순히 같은 배경이 세 번 이어졌다는 이유만으로 바꾸지 않는다.

## 금지

- 제목 밑줄과 장식용 컬러 바
- 과도한 카드·배지·버튼형 UI
- 출처 없는 외부 수치·인용
- 텍스트를 이미지로 굳히기
- 겹침·잘림 경고 무시
